Introduction and Instructions

This server has been preconfigured with PostgreSQL.

PostgreSQL will be used to store Izenda's Configuration Database by default. You can use any supported database for configuration storage just by changing the connection string on the Izenda System Settings page.

You can view the Izenda application here. The initial credentials are below:

User Name: IzendaAdmin

Password: IzendaAdmin123

Your next steps are documented System DB and License.

Please follow the instructions in the article linked above. Please note that Izenda is already connected to its configuration database. The default postgres credentials are below:

Password: Izenda@123

Making the Izenda Application Publicly Accessible

This Izenda instance is only accessible via localhost. If you would like to make the instance publicly accessible, please follow the steps below:

Please check with your network administrators before commiting any changes to your network security.

  • 1

    Obtain the public IP/Domain Name of your provisioned VM from your Azure Portal

  • 2

    Open the izenda_config.js file found in this directory: C:\inetpub\wwwroot\Izenda\App

  • 3

    Edit the WebApiUrl value here to point to the correct address for your api: Ex. http://[Your Public VM IP]:81/api/

  • 4

    In your Azure Portal, you will have to configure the appropriate networking protocols to allow for outside traffic.

    • The UI site should be available remotely to any users who are planning on leveraging the site.
    • The API site needs to be accessible by the UI site to submit requests to the back-end of the application.

  • 5

    In the VM itself, you will need to add Inbound rules to your firewall to allow connections to ports 80 and 81.

    • Open the Windows Firewall with Advanced Security tool.
    • Click on 'New Rule...'
    • Choose the Port option.
    • In the 'Specified local ports:' section input the value '80-81'
    • The API site needs to be accessible by the UI site to submit requests to the back-end of the application.
    • Leave the default action values that are set and click Next.
    • Leave the default Profile options that are set and click Next.
    • Name the rule and hit Finish.

PostgreSQL Troubleshooting:

Depending on the size of the VM and available resources, several errors may occurr with your PostgreSQL Server


  • 1

    Unauthorized Error:

    • This may occur if the stored database information doesn't match the new server configuration.
    • To resolve this, navigate to C:\Users\[your username]\AppData\Roaming\pgAdmin and delete the pgadmin4.db file

  • 2

    500 Internal Server Error:

    • This means that you cannot connect to your PostgreSQL server or it is not currently running.
    • If your VM was low on memory, it's possible that the server did not actually start running. In order to manually start the server, follow these steps:
      1. Open a command prompt as an administrator.
      2. Navigate to the bin directory for your PostgreSQL resources: Ex. C:\Program Files\PostgreSQL\11\bin
      3. Run the following command to start the server: pg_ctl -D "C:\Program Files\PostgreSQL\11\data" start
